# PRODUCT_SPEC.md — 产品规格

## 项目身份

| 字段 | 值 |
|------|-----|
| 项目名称 | 文字里的远方 |
| 副标题 | 三毛《万水千山走遍》文学场景地图 |
| 产品类型 | 静态文学地图网页 |
| 目标用户 | 读过或想读三毛的人、文学爱好者、喜欢安静浏览的人 |

## 产品定位

这是一个**文学场景地图**，核心是"地图"而非"手账"或"攻略"。

- **是**：一张可以点击的文学地图，把书中的旅行地点落到地图上
- **不是**：旅游攻略网站、百科词条、三毛旅行手账、读书笔记

"旧纸感、明信片、旅行手账"是 UI 风格，用于营造文学氛围，不是产品名称。

## 功能范围

### 第一版已实现
1. 首页首屏（标题、副标题、两个按钮）
2. SVG 复古世界地图 + 6 个地点标记
3. 点击地点查看地点信息
4. 小场景卡片网格（三列/两列/一列响应式）
5. 场景详情弹窗
6. 随机抵达一站（全局随机 + 地点内随机）
7. 关于区域

### 明确不会做
- 登录、收藏、评论、搜索
- 后端、数据库、AI 接口
- 真实地图 SDK
- 旅游攻略内容
- 百科词条
- 后台管理

## 页面结构

1. **HeroSection** — 首屏，标题 + 两个按钮
2. **LiteraryWorldMap** — SVG 世界地图 + 6 个地点标记
3. **PlacePanel** — 当前选中地点的信息 + 小场景快捷入口
4. **SceneGrid** — 当前地点的场景卡片墙
5. **SceneDetailModal** — 场景详情弹窗
6. **RandomSceneModal** — 随机明信片弹窗
7. **AboutSection** — 关于说明

## 数据模型

见 `/src/data/places.ts` 和 `docs/DATA_GUIDE.md`。

## 技术限制

- 纯前端静态网页
- 无后端、无数据库、无第三方地图服务
- 图片均为静态资源或 CSS/SVG 占位图
- 不依赖外部 API

## 迭代原则

- 保持"文学地图"定位不变
- 不在 UI 打磨中偏离"地图"核心
- 不引入新依赖
- 优先保证可读性和稳定性
